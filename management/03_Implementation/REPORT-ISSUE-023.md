# 03_実装報告書: ISSUE-023 新規記録追加ボタンの再表示バグ修正レポート

## 概要
タブ切り替え時および別顧客カルテの選択時に、廃止したはずの新規施術記録追加ボタン（「＋ この日の記録を追加」）が画面に再露出してしまう表示不具合を修正しました。

---

## 背景と根本原因分析
v1.9.0（ISSUE-022）において、新規の記録追加動線をカレンダービューへ一本化するため、カルテ詳細画面のボタンを HTML から隠し、イベントハンドラも削除しました。

しかし、顧客詳細ビューがレンダリングされる際、タブが「個人情報以外」に切り替わるたびに `ui.js` の `renderTabContent()` 内で以下の制御が走っていました：
```javascript
        if (activeTab === 'personal-info') {
            addRecordAction.style.display = 'none';
        } else {
            addRecordAction.style.display = 'block';
        }
```
この `block` への動的なインラインスタイル上書き処理が残っていたため、別のタブ（来店回数など）を表示した段階でボタンが再露出していました。イベントハンドラ自体は削除されていたため、画面上には「クリックしても何も機能しないボタン」が残存する形となっていました。

---

## 修正内容

### JS表示制御の修正 (`product/js/app/ui.js`)
`renderTabContent` 内で `display = 'block'` に上書きしていた条件分岐を削除し、カレンダー直結化に伴って常に非表示（`display: 'none'`）を維持するように修正しました。

```diff
-        // 個人情報タブ以外ではレコード追加アクションを表示
-        if (activeTab === 'personal-info') {
-            addRecordAction.style.display = 'none';
-        } else {
-            addRecordAction.style.display = 'block';
-        }
+        // [PATCH v1.9.1] カレンダー直結化に伴い、新規記録追加ボタンは常に非表示にする
+        if (addRecordAction) {
+            addRecordAction.style.display = 'none';
+        }
```

---

## 検証結果
- **顧客切り替え検証**: 山田花子、佐藤健太など、別々の顧客カルテを開いた際にいずれも「＋ この日の記録を追加」ボタンが表示されないことを確認。
- **タブ切り替え検証**: 「来店回数」「施術内容」「金額」「個人情報」の各タブを行き来した際にも、一切ボタンが再表示されないことを確認。
- **過去の施術記録の編集**: 履歴カード上の「✏️ 編集」ボタンからの更新モーダル（`openRecordEditor`）の起動および更新処理は問題なく動作することを確認。
- **新規追加の動作**: カレンダー直結フォームからの追加が問題なく動作し、ソウルカラードット・カルテ詳細の履歴に即時反映されることを確認。
