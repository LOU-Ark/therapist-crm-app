# ISSUE-014: カレンダーから「顧客一覧」タブへ戻ると画面が真っ白になるバグ

## 概要
- **発生日**: 2026-07-20
- **分類**: PATCH (v1.5.8)
- **報告者**: ユーザー様（スクリーンショット3点によるご報告）

## 症状
カレンダービューを表示している状態で「👥 顧客一覧」タブをクリックすると、検索バーも顧客カードも表示されず、コンテンツ領域が完全な空白になる。ブラウザをリロードすると正常な顧客一覧が復帰する。

再現手順:
1. 顧客一覧からカルテ詳細を開く（またはカレンダーの施術記録カードから開く / ISSUE-013の導線）
2. 「🗓 カレンダー」タブに切り替える
3. 「👥 顧客一覧」タブに切り替える → **画面が真っ白**
4. リロード → 正常表示

## 根本原因
`.workspace-grid` のビュー状態クラス（`detail-mode` / `calendar-mode`）が**排他管理されていなかった**ことが原因。

「顧客一覧」タブのクリックハンドラは `calendar-mode` のみを削除し、`detail-mode` を削除していなかった。そのため以下が同時に成立し、左右どちらのカラムも表示されない状態になった。

| 要素 | 状態 | 理由 |
|---|---|---|
| `.workspace-column-left`（検索＋一覧） | 非表示 | `.workspace-grid.detail-mode .workspace-column-left { display: none; }` が残存 |
| `.workspace-column-right` 内 `#customer-detail-view`（カルテ） | 非表示 | 直前の「カレンダー」ハンドラが `style.display = 'none'` を設定済み |

結果としてワークスペース全体が空になる。リロードで直るのは、DOM再構築によりクラスとインラインスタイルが初期化されるため。

なお ISSUE-013 の修正で `calendar-mode` の解除漏れは塞いだが、**同種の「状態クラス解除漏れ」がもう一方（`detail-mode`）にも存在していた**ことを見落としていた。

## 修正内容
`product/js/app/ui.js` のビュー切り替えハンドラを、状態クラスが常に排他になるよう修正。

1. `btn-view-list`（顧客一覧）ハンドラ: `detail-mode` の削除、`#customer-detail-view` の非表示、`selectedCustomerId` のリセット、一覧の再描画を追加。
2. `btn-view-calendar`（カレンダー）ハンドラ: `detail-mode` を明示的に削除（CSS記述順に依存した打ち消しを廃止）。
