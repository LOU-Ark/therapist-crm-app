# ISSUE-015: カレンダーから顧客一覧へ戻るとカードグリッドが縦積みリストに崩れるバグ

## 概要
- **発生日**: 2026-07-20
- **分類**: PATCH (v1.5.9)
- **報告者**: ユーザー様（「これはキャッシュを読み込んでいるのでしょうか？」とのご質問付き）

## 症状
カレンダービューから「👥 顧客一覧」タブへ戻ると、本来2列以上のタイル状グリッドで並ぶべき顧客カード（C-0001 山田花子 / C-0002 佐藤健太）が、**画面全幅の横長カードとして縦一列に積み上がって**表示される。ブラウザをリロードすると正しいグリッド表示に戻る。

※ ユーザー様のご質問への回答：**キャッシュではありません。** 実行時にJavaScriptが付与したインラインスタイルが残留していたことが原因で、リロードでDOMが再構築されるとインラインスタイルが消えるため正常化していました。

## 根本原因
CSS では顧客一覧コンテナがグリッドとして定義されている。

```css
/* product/css/index.css:133-138 */
.customer-list {
    display: grid;
    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
    gap: 16px;
    width: 100%;
}
```

しかし JavaScript のビュー切り替え処理が、表示/非表示の制御のために **インラインで `display: block` を代入**していた。

```js
customerListContainer.style.display = 'block';   // ← grid を破壊
```

インラインスタイルは通常のCSSセレクタより優先されるため `display: grid` が無効化され、`grid-template-columns` も効かなくなり、子カードが通常のブロックフローで縦積みになった。

初期表示時はこの代入が走らないためグリッドが保たれており、**「カレンダーを経由して一覧へ戻る」経路でのみ**発現していた。ISSUE-014 修正以前はこの経路が空白画面になっていたため、本バグは到達不能で顕在化していなかった。

## 修正内容
`product/js/app/ui.js` の2箇所で、`style.display = 'block'` を `style.display = ''`（インライン指定の除去）に変更し、CSS本来の `display: grid` に復帰させた。

- 「顧客一覧」タブのクリックハンドラ
- カレンダーの施術記録カードからカルテへ遷移するハンドラ
