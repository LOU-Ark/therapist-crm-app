# ISSUE-017: スマホ画面でカレンダーが横に見切れるバグ

## 概要
- **発生日**: 2026-07-20
- **分類**: PATCH (v1.5.10)
- **報告者**: ユーザー様

## 症状
スマートフォンの画面幅（375〜390px）でカレンダービューを表示すると、**土曜日の列と月送りの「→」ボタンが画面右端で見切れる**。`body` に `overflow-x: hidden` が指定されているため横スクロールもできず、見切れた部分に一切アクセスできない状態だった。

## 根本原因

### 主因: `aspect-ratio` による最小サイズの軸間転写

```css
.calendar-day {
    aspect-ratio: 1;
    min-height: 48px;   /* ← これが「最小幅48px」としても効く */
}
```

CSS の `aspect-ratio` は **min/max 制約を軸間で転写する**仕様のため、`min-height: 48px` が幅の最小値 48px としても作用していた。結果、カレンダーグリッドに以下の最小幅が生まれていた。

```
セル 48px × 7列 + gap 8px × 6 = 384px
+ .calendar-view-section の padding 20px × 2 + border = 42px
+ main の padding 16px × 2                            = 32px
------------------------------------------------------------
合計 458px  >  スマホ幅 390px
```

### 副因1: グリッドトラックの暗黙の最小幅
`grid-template-columns: repeat(7, 1fr)` の `1fr` は暗黙に `min-width: auto` を持つため、トラックがセルの内容幅より狭くなれない。上記の転写と合わせて、縮小の余地が二重に塞がれていた。

### 副因2: レスポンシブ指定が皆無
`product/css/index.css` に **`@media` クエリが1つも存在しなかった**。CSS冒頭に「レイアウト (スマホファースト)」というコメントがあるにもかかわらず、実際には画面幅に応じた調整が一切実装されていなかった。

## 修正内容
1. `.calendar-day` の `min-height: 48px` を `min-width: 0; min-height: 0;` に変更し、転写される最小幅を解消。
2. `.calendar-grid` を `repeat(7, minmax(0, 1fr))` に変更し、トラックの暗黙の最小幅を解除。`width: 100%` を明示。
3. `@media (max-width: 640px)` を新設し、`main` / `.calendar-view-section` の padding と grid の gap を詰めてセルの表示サイズを確保。

## 検証
ヘッドレスChromeで幅390pxのビューポートを再現し、修正前後を比較。

| | 結果 |
|---|---|
| 修正前 | 土曜日の列と「→」ボタンが見切れる |
| 修正後 | 日〜土の7列と「→」ボタンがすべて画面内に収まる |
| 計測値 | `viewport=390` / `bodyScrollW=390` / **はみ出し要素 0件** |

## 修正ファイル
- `product/css/index.css`（629行目付近 / 648行目付近 / 末尾に @media を追加）
