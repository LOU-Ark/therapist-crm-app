# ISSUE-019: 既存の施術記録で「訴え」「処方」「メモ」が表示されないバグ

## 概要
- **発生日**: 2026-07-21
- **分類**: PATCH (v1.6.0 → v1.6.1)
- **報告者**: ユーザー様（「クライアント側の訴えや処方がありません」とのご指摘）

## 症状
ISSUE-018 で「施術内容」タブに訴え・処方・メモを表示するようにしたにもかかわらず、実際の画面では施術メニュー名しか表示されず、3項目が現れない。

## 根本原因

### 直接原因: 空文字と条件付きレンダリングの組み合わせ

訴え・処方・メモのフィールドが存在しなかった時代に作られた記録は、スキーママイグレーションによって**空文字 `""` で埋められる**。

```js
// data.js
if (r.clientComplaint === undefined) { r.clientComplaint = ""; updated = true; }
if (r.prescription === undefined)    { r.prescription = "";    updated = true; }
if (r.therapistNote === undefined)   { r.therapistNote = "";   updated = true; }
```

一方、描画側は「値があるときだけ行を出す」条件付きレンダリングだった。

```js
${r.clientComplaint ? `<div>訴え: ${r.clientComplaint}</div>` : ''}
```

空文字は falsy のため**行ごと消える**。結果、ユーザー様の端末では項目自体が存在しないように見えていた。

### 再現による確認

旧バージョン相当のデータ（3フィールドを持たない記録）を localStorage に投入して再現したところ、以下のとおりであった。

```
storedRecord : { date, type, amount, time, clientComplaint:"", prescription:"", therapistNote:"" }
renderedText : "2026-07-10 (14:00 - 15:00) アロママッサージ 60分"
```

### なぜ検証をすり抜けたか

ISSUE-018 の検証を**デモデータ（3項目に値が入っている）のみ**で行い、既存ユーザーの実データに相当する「空文字」のケースを試さなかった。ISSUE-018 で自ら追加した **[V-92] の「入力・保存・表示の3点セット確認」を、空データの経路で実施していなかった**ことになる。

## 修正内容
未記入でも項目名は必ず表示し、値のみをプレースホルダ「未記入」に置き換える方式へ変更した。あわせて、来店回数タブと施術内容タブで重複していた描画コードを `buildRecordDetailsHtml()` に共通化した。

## 修正ファイル
- `product/js/app/ui.js`

## 次の対応（合意済み）
過去の施術記録を後から編集する機能が存在しないため、既存記録に訴え・処方を追記できない。この編集機能は別途対応する（ISSUE-020 予定）。
