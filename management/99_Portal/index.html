v1.5.10 - スマホ画面でカレンダーが横に見切れるバグの修正 (ISSUE-017)

スマホ幅（375〜390px）で土曜日の列と月送りボタンが右端で見切れていました。aspect-ratio は min/max 制約を軸間で転写するため、.calendar-day の min-height: 48px が「最小幅48px」としても作用し、カレンダー全体に458pxの最小幅が生まれていたことが原因です。最小サイズの解除・minmax(0, 1fr) 化に加え、これまで1つも存在しなかった @media クエリを新設しました。ヘッドレスブラウザで390px幅を再現し、はみ出し要素0件を実測確認済みです。

[CHORE] v1.5.9 - Vercelデプロイ対象をアプリ単体に限定・トップページ化 (ISSUE-016)

Vercelの配信対象を product/（顧客管理アプリ）のみに限定し、顧客一覧画面がトップページ（/）に来るよう設定を変更しました。.vercelignore で管理ポータル・開発用ランチパッド・ブリッジサーバを除外し、vercel.json に outputDirectory: "product" を追加しています。あわせて、本番では404となっていたフローティングチャット（.agent/ 配下を参照＝Git管理外）を product/js/vendor/ へ同梱し、アプリを自己完結させました。設定変更のためバージョンは v1.5.9 据え置きです。管理ポータルは今後ローカルでのみ閲覧します。

v1.5.9 - インラインスタイルによる顧客カードグリッド崩れの修正 (ISSUE-015)

カレンダーから顧客一覧へ戻ると、カードがタイル状グリッドではなく全幅の縦積みリストに崩れていました。JS が表示制御のために style.display = 'block' をインライン付与しており、CSS の .customer-list { display: grid } を優先度で上書きしていたことが原因です。style.display = ''（インライン指定の除去）に変更し、CSS本来のグリッドへ復帰させました。キャッシュではなくインラインスタイルの残留が「リロードで直る」挙動を生んでいました。
